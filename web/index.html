GotBox

Cross-machine file sync backed by your own GitHub account. Edit a file and it's automatically committed, versioned, and pushed to a private repo you own — changes from your other machines flow back on their own. No proprietary cloud, no subscription, no file-size surprises.

How it works

1

Pick a sync folder

Choose one root folder (default ~/GotBox). It becomes the working copy of a private .gotbox repo.

2

Just edit files

GotBox watches for changes, batches a burst of saves, then commits & pushes automatically in the background.

3

Other machines pull

Sign in with the same account elsewhere; a cheap remote check pulls new commits as soon as they appear.

4

Stays lean forever

History is capped by a rolling squash and git maintenance runs on its own, so repos never grow without bound.

Features

🔄

Automatic two-way sync

Local edits are committed and pushed; remote changes are pulled and merged — all silently in the background.

🔒

Backed by private git

Your files live in repos you own on GitHub (or a self-hosted git server). Full version history comes for free.

💻

Multi-machine

Set up the same account on another computer and your files appear there; edits propagate both ways.

🧩

Link projects as submodules

Keep distinct repos under one root, each with its own local folder name, each syncing independently.

🛟

Never loses data

If a file changed on two machines, GotBox keeps both versions and alerts you to resolve it.

📦

Handles large files

Files past a size threshold are auto-tracked with Git LFS, so they don't hit GitHub's 100 MB push limit.

🔑

Secure credentials

Your GitHub token lives only in the OS keychain (libsecret / Keychain / DPAPI) — never in a plain config file.

🔔

Desktop notifications

A quiet bubble tells you what synced — file names for a few, a count for many — and flags conflicts or errors.

🏷️

File-manager badges

Windows & macOS: TortoiseGit-style overlays show per-file synced / modified / conflict status in Explorer & Finder.

Tray icon states

The tray icon is the GotBox "G-in-a-box", tinted to show the aggregate state of all your repos.

Synced

Everything is committed, pushed, and up to date.

Syncing

Committing, pushing, or pulling changes right now.

Conflict

A file changed on two machines; both kept — open Status to resolve.

Error

A repo could not sync; open Status for details.

Idle / Paused

Nothing to do, or syncing is paused.

Offline

No network; changes are queued and sync resumes automatically.

Get started

1

Install

Grab a prebuilt installer for your OS from the Releases page — a .deb (Linux), setup .exe (Windows), or .dmg (macOS) — or build from source with Lazarus/FPC.

2

Create a GitHub token

Make a classic personal access token with the repo scope so GotBox can create and push to private repos. You paste it once — it's saved to your OS keychain.

3

Sign in & pick a folder

On first launch, enter your GitHub username and token, then confirm the sync folder (defaults to ~/GotBox). GotBox creates the private .gotbox repo automatically the first time content appears.

4

Add a second machine

Install GotBox on another computer, sign in with the same account, and point it at a folder. GotBox clones your existing .gotbox (and its linked submodules) and keeps everything in sync.

# build from source (needs Lazarus/FPC + git)
make            # build the tray app
make run        # build and launch
make install    # install binary + desktop entry + icons
make autostart  # also start GotBox on login

Commit debounceQuiet time (ms) after the last save before committing — batches a burst into one commit.10000
Pull intervalHow often (s) to check the remote. A cheap ls-remote probe runs each interval; a full fetch only when the remote actually moved.15
MaintenanceRun git gc / repack after this many commits.25
LFS thresholdAuto-track files at/above this size (MB) with Git LFS; 0 disables.95
Ignore patternsGlobs never synced..git, *.tmp, *~
